# Database Schema

## Collection: `articles`

The `articles` collection will store all news articles uploaded by journalists. The schema is designed to closely match the frontend's `ArticleEntity` to make data parsing straightforward while adapting to NoSQL conventions.

### Document Structure

| Field         | Firestore Type | Description |
| :---          | :---           | :--- |
| `id`          | String         | Unique document ID (auto-generated by Firestore). |
| `author`      | String         | The name of the journalist who wrote the article. |
| `title`       | String         | The headline of the article. |
| `description` | String         | A brief summary of the article. |
| `url`         | String         | An optional external URL reference for the article. |
| `urlToImage`  | String         | Reference URL to the thumbnail stored in Firebase Cloud Storage (`media/articles/`). |
| `publishedAt` | Timestamp      | The date and time the article was created/published. |
| `content`     | String         | The full body content of the article. |

### Example Document JSON

```json
{
  "author": "Miguel García",
  "title": "Flutter's New Clean Architecture Paradigm",
  "description": "An in-depth look at how Symmetry implements Clean Architecture.",
  "url": "",
  "urlToImage": "https://firebasestorage.googleapis.com/v0/b/YOUR-PROJECT/o/media%2Farticles%2Fflutter_clean.jpg?alt=media",
  "publishedAt": "2026-03-12T22:20:00.000Z",
  "content": "Full content of the article goes here..."
}
```
